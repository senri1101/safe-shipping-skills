# Project Profile

レビューは必ずこのプロファイルから始める。入力が曖昧でも空欄のまま進めず、repo・仕様・設定・ユーザー説明から仮置きする。

## テンプレート

```md
## Project Profile

| Item | Value | Evidence / Notes |
| --- | --- | --- |
| Project Type | 使い捨て / 継続運用 | |
| Exposure | 個人用 / 社内 / 限定公開 / 一般公開 | |
| Data Class | なし / 内部情報 / 個人情報 / 決済・規制対象 | |
| Auth / Authz / Multi-tenant | なし / あり | |
| Variable-cost Services | なし / LLM / DB Read / Image Gen / CDN / Other | |
| Runtime Base | SaaS builder / PaaS / IaaS / ローカルのみ | |
| AI / LLM Usage | なし / あり | |
| Current Stage | 企画 / 設計 / 実装 / リリース前 / 運用中 | |
```

## 項目ごとの判定ガイド

### Project Type
- `使い捨て`: 数日から数週間で廃棄予定。継続運用や第三者利用を想定しない
- `継続運用`: 月単位で保守、改修、ユーザーサポートが発生する
- 不明なら `継続運用` 扱いに寄せる

### Exposure
- `個人用`: 開発者本人だけが使う
- `社内`: 同一組織内の利用者に限定
- `限定公開`: 顧客、βユーザー、招待制など
- `一般公開`: 誰でも利用できる、または URL を知れば到達できる
- 不明なら 1 段強い公開範囲で仮置きする

### Data Class
- `なし`: 永続データを持たず、利用者入力も機微情報を含まない
- `内部情報`: 組織外に出してはいけないが、法規制対象ではない
- `個人情報`: 氏名、メール、住所、社員番号、位置情報、行動履歴など
- `決済・規制対象`: 決済、医療、金融、法務、年齢制限、監査対象データなど
- 不明なら最低でも `内部情報` とする

### Auth / Authz / Multi-tenant
- `なし`: 完全な静的公開物など
- `あり`: ログイン、役割、所有者確認、テナント分離のいずれかがある
- ログインだけある場合も `あり`

### Variable-cost Services
- 従量課金の対象を列挙する
- LLM、画像生成、DB 読み取り、CDN、外部 API、キュー、メール、SMS を見落とさない
- 不明なら README、env、SDK、課金ダッシュボード連携コードを探す

### Runtime Base
- `SaaS builder`: Lovable / Bolt 系の統合基盤
- `PaaS`: Vercel / Firebase / Supabase / Heroku など
- `IaaS`: AWS / GCP / Azure を生で組む、VM / VPC / SG / IAM を直接扱う
- `ローカルのみ`: 配布しない、社内共有もしない
- IaaS なら Security / Backup / Logging の要求水準を上げる

### AI / LLM Usage
- 生成、分類、要約、エージェント実行、コード生成支援を含む
- 社内データを外部 AI に送る場合は、承認済みツールかどうかも記録する

### Current Stage
- `企画`: まだ作る前。要件、用途、法務、コスト、公開範囲を固める段階
- `設計`: データ、認可、トランザクション、環境分離を固める段階
- `実装`: コード、テスト、依存関係、CI を見る段階
- `リリース前`: 観測、予算、バックアップ、ロールバックを確認する段階
- `運用中`: 監視、再発防止、トレンド確認、権限棚卸しを見る段階

## 事実の拾い方

- repo がある場合は、README、env サンプル、lock file、CI、infra 定義、auth 周辺、migration、logging 設定を先に確認する
- 仕様しかない場合は、利用者、データ、公開範囲、外部連携、失敗時の影響を先に確認する
- 不足情報は `Unknown` のまま放置せず、`仮置き値 + 理由` を書く

## 不明時のデフォルト

- 公開範囲が曖昧なら 1 段強く扱う
- 個人情報か不明なら `内部情報` 以上で扱う
- 認可の証拠がなければ `Authz 未確認`
- バックアップの証拠がなければ `Backup なし`
- 予算アラートの証拠がなければ `Cost Guardrail なし`
